Training Overview: Migrate SQL Server workloads to Azure SQL Database

This intermediate-level training prepares learners to assess, plan, and execute migrations of SQL Server workloads to Azure SQL Database using both IaaS and PaaS strategies. Participants will design migration strategies tailored to workload requirements, assess database readiness and compatibility, provision and configure Azure SQL resources, perform migrations to Azure VMs, Azure SQL Database, and Azure SQL Managed Instance, and carry out post-migration tasks such as configuration and disaster recovery.

Before enrolling in Migrate SQL Server Workloads to Azure SQL Database, it is ideal for participants to be database administrators, data engineers, or IT professionals with SQL Server experience and familiarity with the Azure portal.

Module Breakdown:

  • Design a SQL Server Migration Strategy: Define migration goals, evaluate options (IaaS vs PaaS), and select the right Azure SQL target.

  • Assess SQL Server Databases for Migration to Azure SQL: Use tools like DMA and DMS to evaluate schema compatibility and performance readiness.

  • Deploy IaaS Solutions with Azure SQL: Provision Azure VMs, configure SQL Server, and prepare for lift-and-shift migrations.

  • Deploy PaaS Solutions with Azure SQL: Set up Azure SQL Database and Managed Instance, configure networking and security.

  • Migrate SQL Server Workloads to Azure Virtual Machine: Execute migration using backup/restore, manual copy, or DMS for VM-based targets.

  • Migrate SQL Server Workloads to Azure SQL Database: Perform schema and data migration to PaaS targets using DMS and post-migration validation.

  • Migrate SQL Server Workloads to Azure SQL Managed Instance: Migrate complex workloads with instance-level features using DMS and configuration tools.

  • Post-Migration Configuration & Disaster Recovery: Configure backups, high availability, and disaster recovery for long-term resilience.
